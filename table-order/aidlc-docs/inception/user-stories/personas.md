# 테이블오더 서비스 - 사용자 페르소나

---

## Persona 1: 고객 (Customer)

### 프로필
| 항목 | 내용 |
|------|------|
| **이름** | 김민수 (대표 페르소나) |
| **역할** | 매장 방문 고객 |
| **연령대** | 20~60대 (전 연령) |
| **기술 수준** | 다양 (스마트폰 기본 사용 가능 수준) |
| **사용 기기** | 테이블 고정 태블릿 (브라우저 기반) |

### 목표 (Goals)
- 대기 없이 빠르게 주문하고 싶다
- 메뉴를 쉽게 탐색하고 선택하고 싶다
- 주문 상태를 확인하고 싶다
- 추가 주문을 편하게 하고 싶다

### 동기 (Motivations)
- 직원 호출 없이 자유롭게 주문
- 메뉴 사진과 설명을 보고 선택
- 주문 실수를 줄이고 싶음 (장바구니에서 확인)

### 불편사항 (Pain Points)
- 복잡한 인터페이스 (로그인, 회원가입 등)
- 느린 반응 속도
- 주문이 제대로 들어갔는지 불안함
- 이전 손님의 주문 내역이 보이는 것

### 사용 시나리오
1. 테이블에 앉으면 태블릿이 이미 메뉴 화면을 보여줌
2. 카테고리별로 메뉴를 탐색하고 장바구니에 담음
3. 장바구니에서 수량 조절 후 주문 확정
4. 주문 상태를 확인하며 음식 대기
5. 추가 주문이 필요하면 같은 과정 반복

## Persona 2: 매장 관리자 (Admin)

### 프로필
| 항목 | 내용 |
|------|------|
| **이름** | 박서연 (대표 페르소나) |
| **역할** | 매장 운영자 / 점주 |
| **연령대** | 30~50대 |
| **기술 수준** | 중간 (PC/태블릿 기본 사용 가능) |
| **사용 기기** | PC 또는 태블릿 (관리자 전용 화면) |

### 목표 (Goals)
- 들어오는 주문을 실시간으로 파악하고 싶다
- 테이블별 주문 현황을 한눈에 보고 싶다
- 주문 상태를 효율적으로 관리하고 싶다
- 테이블 회전율을 관리하고 싶다 (이용 완료 처리)

### 동기 (Motivations)
- 인건비 절감 (주문 접수 인력 불필요)
- 주문 누락 방지
- 매장 운영 효율성 극대화
- 과거 주문 내역으로 매출 파악

### 불편사항 (Pain Points)
- 주문이 밀릴 때 놓치는 것
- 잘못된 주문을 수정하기 어려운 것
- 테이블 상태 파악이 어려운 것
- 이전 손님 주문이 남아있는 것

### 사용 시나리오
1. 매장 오픈 시 관리자 화면 로그인
2. 대시보드에서 테이블별 주문 현황 실시간 모니터링
3. 신규 주문 확인 → 준비중으로 상태 변경 → 완료 처리
4. 잘못된 주문 삭제 (직권 수정)
5. 고객 퇴장 시 테이블 이용 완료 처리
6. 필요 시 과거 주문 내역 조회
