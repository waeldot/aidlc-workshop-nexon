# 테이블오더 서비스 - 요구사항 문서

## 인텐트 분석 요약

| 항목 | 내용 |
|------|------|
| **요청 유형** | 신규 프로젝트 (New Project / Greenfield) |
| **범위** | 시스템 전체 — 고객 UI, 관리자 UI, 백엔드 API, 데이터베이스 |
| **복잡도** | 복합적 (Moderate-Complex) |
| **핵심 특성** | 실시간 통신(SSE), 세션 기반 인증, 멀티 역할 UI, 테이블 세션 라이프사이클 관리 |

## 기술 스택 결정사항

| 영역 | 선택 | 비고 |
|------|------|------|
| **백엔드** | Go | 경량 고성능 API 서버 |
| **프론트엔드** | Vue.js (TypeScript) | 고객 앱 + 관리자 앱 별도 구성 |
| **데이터베이스** | SQLite | 단일 매장, 소규모 운영에 적합 |
| **배포** | Docker 컨테이너 | 환경 무관 배포 |
| **레포지토리 구조** | 모노레포 | 백엔드 + 프론트엔드 함께 |
| **PBT 프레임워크** | rapid (Go 백엔드), fast-check (Vue 프론트엔드) | PBT 확장 활성화됨 |

## 아키텍처 결정사항

- **프론트엔드 앱**: 고객용(customer-app)과 관리자용(admin-app) **별도 Vue.js 앱** 2개
- **백엔드**: 단일 Go API 서버 (REST + SSE)
- **실시간 통신**: SSE (Server-Sent Events) — 관리자 화면 및 고객 화면 모두 적용
- **이미지 관리**: 이미지 URL만 저장 (외부 링크)
- **인증 토큰 저장**: localStorage (태블릿 자동 로그인)

## 기능 요구사항

### FR-01: 테이블 태블릿 자동 로그인 및 세션 관리 (고객)

**목적**: 고객이 별도 로그인 없이 즉시 주문 가능하도록 태블릿 기기를 특정 테이블에 귀속

**요구사항**:
- 관리자가 최초 1회 태블릿에서 매장 식별자, 테이블 번호, 테이블 비밀번호 입력
- 입력 정보를 localStorage에 저장
- 이후 앱 실행 시 저장된 정보로 자동 로그인 (서버 인증 토큰 발급)
- 인증 실패 시 초기 설정 화면으로 이동

---

### FR-02: 메뉴 조회 및 탐색 (고객)

**요구사항**:
- 앱 기본 화면은 메뉴 화면
- 카테고리별 메뉴 분류 및 표시
- 메뉴 카드: 메뉴명, 가격, 설명, 이미지 URL 표시
- 카테고리 간 빠른 이동 (탭 또는 사이드바)
- 터치 친화적 UI (버튼 최소 44×44px)

---

### FR-03: 장바구니 관리 (고객)

**요구사항**:
- 메뉴 추가 / 삭제
- 수량 증가 / 감소
- 총 금액 실시간 계산
- 장바구니 전체 비우기
- localStorage에 임시 저장 (새로고침 후에도 유지)
- 서버 전송은 주문 확정 시에만 수행

---

### FR-04: 주문 생성 (고객)

**요구사항**:
- 주문 내역 최종 확인 화면
- 주문 확정 버튼
- 주문 성공 시:
  - 주문 번호 표시 (5초 후 자동 리다이렉트)
  - 장바구니 자동 비우기
  - 메뉴 화면으로 이동
- 주문 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보 포함: 매장 식별자, 테이블 식별자, 세션 ID, 메뉴 목록(메뉴명/수량/단가), 총 금액

---

### FR-05: 주문 내역 조회 (고객)

**요구사항**:
- 현재 테이블 세션의 주문만 표시 (이전 세션 제외)
- 주문 시간 순 정렬
- 주문별 표시: 주문 번호, 주문 시각, 메뉴 및 수량, 금액, 주문 상태
- 주문 상태 실시간 업데이트 (SSE)
- 페이지네이션 또는 무한 스크롤

---

### FR-06: 관리자 인증

**요구사항**:
- 매장 식별자 + 사용자명 + 비밀번호 로그인
- JWT 토큰 기반 인증
- 세션 16시간 유지, 브라우저 새로고침 후에도 유지
- 16시간 후 자동 로그아웃
- 비밀번호 bcrypt 해싱 저장
- 로그인 시도 횟수 제한 (브루트포스 방지)

---

### FR-07: 실시간 주문 모니터링 (관리자)

**요구사항**:
- SSE 기반 실시간 주문 업데이트 (2초 이내 표시)
- 그리드/대시보드 레이아웃 — 테이블별 카드 형태
- 각 테이블 카드: 총 주문액, 최신 주문 n개 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경: 대기중 → 준비중 → 완료
- 신규 주문 시각적 강조 (색상 변경, 애니메이션)
- 테이블별 필터링

---

### FR-08: 테이블 관리 (관리자)

**요구사항**:

**1. 태블릿 초기 설정**:
- 테이블 번호 및 비밀번호 설정
- 16시간 세션 생성

**2. 주문 삭제 (직권 수정)**:
- 특정 주문 삭제 (확인 팝업)
- 삭제 후 테이블 총 주문액 재계산
- 성공/실패 피드백

**3. 테이블 세션 처리 (이용 완료)**:
- 확인 팝업 후 세션 종료
- 해당 세션 주문 내역을 과거 이력(OrderHistory)으로 이동
- 테이블 현재 주문 목록 및 총 주문액 0으로 리셋
- 새 고객이 이전 주문 내역 없이 시작 가능

**4. 과거 주문 내역 조회**:
- 테이블별 과거 주문 목록 (시간 역순)
- 주문 정보: 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각
- 날짜 필터링
- 닫기 버튼으로 대시보드 복귀

---

### FR-09: 메뉴 데이터 (Seed)

**요구사항**:
- MVP에서 메뉴 CRUD UI 미구현
- 메뉴 데이터는 DB 초기 데이터(seed)로 관리
- 카테고리 및 메뉴 항목 포함

## 비기능 요구사항

### NFR-01: 성능
- 주문 API 응답 시간: 500ms 이내
- SSE 이벤트 전달: 2초 이내
- 동시 접속: 단일 매장 기준 최대 10개 테이블 + 관리자 1명

### NFR-02: 보안 (Security Extension 활성화)
- 모든 보안 규칙(SECURITY-01 ~ SECURITY-15) 필수 적용
- 비밀번호 bcrypt 해싱
- JWT 서버 사이드 검증 (모든 요청)
- API 입력값 검증 및 SQL 인젝션 방지 (파라미터화 쿼리)
- HTTP 보안 헤더 (CSP, HSTS, X-Content-Type-Options 등)
- 레이트 리미팅 (공개 API 엔드포인트)
- 구조화된 로깅 (민감 정보 로그 제외)
- CORS 제한 (허용된 오리진만)

### NFR-03: 신뢰성
- 주문 생성 실패 시 장바구니 데이터 보존
- SSE 연결 끊김 시 자동 재연결
- 에러 응답에 내부 정보 노출 금지

### NFR-04: 사용성
- 터치 친화적 UI (최소 44×44px 버튼)
- 카드 형태 메뉴 레이아웃
- 명확한 시각적 계층 구조

### NFR-05: 테스트 (PBT Extension 활성화 — Full)
- 모든 PBT 규칙(PBT-01 ~ PBT-10) 필수 적용
- Go 백엔드: `rapid` 프레임워크
- Vue 프론트엔드: `fast-check` 프레임워크
- 비즈니스 로직, 직렬화, 상태 관리에 속성 기반 테스트 적용
- 예시 기반 테스트와 병행 (PBT-10)

### NFR-06: 배포
- Docker 컨테이너 기반 배포
- 환경 변수로 설정 관리 (하드코딩 금지)

## MVP 범위 요약

### 포함 (In Scope)
| 기능 | 대상 |
|------|------|
| 태블릿 자동 로그인 | 고객 |
| 메뉴 조회 (카테고리별) | 고객 |
| 장바구니 관리 | 고객 |
| 주문 생성 | 고객 |
| 주문 내역 조회 (현재 세션) | 고객 |
| 관리자 인증 (JWT 16시간) | 관리자 |
| 실시간 주문 모니터링 (SSE, 그리드) | 관리자 |
| 주문 상태 변경 (대기중/준비중/완료) | 관리자 |
| 테이블 관리 (초기 설정, 주문 삭제, 이용 완료, 과거 내역) | 관리자 |
| 메뉴 Seed 데이터 | 시스템 |

### 제외 (Out of Scope)
- 결제 처리 및 PG 연동
- 메뉴 CRUD 관리 UI
- 푸시/SMS/이메일 알림
- 주방 기능
- 데이터 분석 / 매출 리포트
- 다국어 지원
- 외부 시스템 연동 (배달 플랫폼, POS 등)
- 이미지 업로드 / 리사이징

## 용어 정의

| 용어 | 정의 |
|------|------|
| **테이블 세션** | 특정 테이블에 고객이 앉아 첫 주문 시작 후 이용 완료 처리까지의 시간 |
| **세션 ID** | 테이블 세션을 식별하는 고유 ID |
| **이용 완료** | 관리자가 테이블 세션을 종료하는 행위 |
| **OrderHistory** | 이용 완료된 세션의 주문 내역을 저장하는 테이블 |
| **태블릿 자동 로그인** | 관리자가 1회 설정 후 태블릿이 자동으로 특정 테이블로 인증되는 기능 |
