# Customer App - NFR Requirements

## 1. 성능 요구사항 (Performance)

### PERF-01: 페이지 로딩
| 지표 | 기준 | 비고 |
|------|------|------|
| First Contentful Paint (FCP) | < 1.5초 | 태블릿 Wi-Fi 환경 |
| Largest Contentful Paint (LCP) | < 2.5초 | 메뉴 이미지 포함 |
| Time to Interactive (TTI) | < 3초 | 사용자 인터랙션 가능 시점 |
| 초기 번들 크기 | < 200KB (gzip) | 라우트별 레이지 로딩 적용 |

### PERF-02: API 응답 처리
| 지표 | 기준 | 비고 |
|------|------|------|
| API 호출 → UI 반영 | < 300ms | 네트워크 제외, 클라이언트 처리 시간 |
| 장바구니 조작 반응 | < 50ms | localStorage 동기화 포함 |
| 카테고리 전환 | < 100ms | 이미 로드된 데이터 필터링 |

### PERF-03: 렌더링 성능
| 지표 | 기준 | 비고 |
|------|------|------|
| 스크롤 FPS | 60fps | 메뉴 목록 스크롤 |
| 애니메이션 FPS | 60fps | 하단 시트, 드로어, 모달 |
| 인터랙션 지연 | < 100ms | 버튼 탭 → 시각적 피드백 |

## 2. 보안 요구사항 (Security)

### SEC-01: 토큰 저장 보안
- localStorage에 JWT 토큰 저장 (태블릿 전용 앱 특성상 허용)
- XSS 공격 방어를 통한 토큰 탈취 방지
- 토큰에 민감 정보 포함 금지 (서버 측 책임)

### SEC-02: XSS 방지
- Vue.js 기본 이스케이핑 활용 (v-html 사용 금지)
- 사용자 입력값 sanitize (설정 화면 입력)
- 서버 응답 데이터도 렌더링 전 이스케이프 확인
- CSP 헤더 적용 (Nginx 설정):
  - `default-src 'self'`
  - `script-src 'self'`
  - `style-src 'self' 'unsafe-inline'` (Tailwind 인라인 스타일)
  - `img-src 'self' https:` (외부 메뉴 이미지 허용)
  - `connect-src 'self'` (API + SSE)

### SEC-03: 입력값 검증
- 매장 식별자: 문자열, 최대 50자, 특수문자 제한
- 테이블 번호: 양의 정수, 1~999 범위
- 비밀번호: 문자열, 최소 1자 (서버에서 강도 검증)
- 모든 입력: trim 처리, HTML 태그 제거

### SEC-04: 통신 보안
- HTTPS 전용 (HTTP 접근 시 리다이렉트)
- API 요청 시 Authorization Bearer 토큰 헤더
- CORS: Backend에서 허용된 오리진만 접근

### SEC-05: 데이터 노출 방지
- 에러 메시지에 내부 정보 노출 금지
- 콘솔 로그에 토큰/비밀번호 출력 금지 (프로덕션)
- 소스맵 프로덕션 빌드에서 제외

## 3. 신뢰성 요구사항 (Reliability)

### REL-01: SSE 연결 안정성
- 무제한 자동 재연결 (지수 백오프: 1s → 2s → 4s → ... → 30s max)
- 연결 상태 내부 추적 (connected / disconnected / reconnecting)
- 재연결 성공 시 최신 데이터 동기화 (주문 목록 재조회)

### REL-02: 네트워크 에러 처리
- 오프라인 미지원 (Service Worker 없음)
- 네트워크 에러 시 사용자 친화적 에러 메시지 표시
- "다시 시도" 버튼 제공 (메뉴 로딩, 주문 내역 로딩)
- 주문 실패 시 장바구니 데이터 보존

### REL-03: 데이터 일관성
- localStorage 읽기 시 JSON 파싱 에러 안전 처리
- 스키마 불일치 시 기본값으로 초기화 (데이터 손실보다 안정성 우선)
- 장바구니 변경 시 즉시 localStorage 동기화 (데이터 유실 방지)

### REL-04: 에러 바운더리
- Vue Error Boundary 패턴 적용
- 컴포넌트 에러가 전체 앱 크래시로 이어지지 않도록 격리
- 에러 발생 시 해당 영역만 에러 상태 표시

## 4. 사용성/접근성 요구사항 (Usability/Accessibility)

### USE-01: 터치 인터페이스
- 모든 인터랙티브 요소: 최소 44×44px 터치 영역
- 버튼 간 최소 간격: 8px
- 스와이프 제스처: 드로어 열기/닫기, 하단 시트 펼치기/접기
- 더블 탭 줌 방지 (viewport meta)

### USE-02: WCAG 2.1 AA 준수
| 기준 | 요구사항 |
|------|----------|
| 색상 대비 | 텍스트 4.5:1, 대형 텍스트 3:1 |
| 포커스 표시 | 모든 인터랙티브 요소에 가시적 포커스 링 |
| 키보드 접근 | Tab 순서 논리적, Enter/Space로 활성화 |
| ARIA | 동적 콘텐츠에 aria-live, 모달에 aria-modal |
| 대체 텍스트 | 메뉴 이미지에 alt 텍스트 (메뉴명) |
| 상태 전달 | 주문 상태 배지에 aria-label (색상만으로 구분 안 함) |

### USE-03: 반응형 레이아웃
- 주 타겟: 태블릿 (768px ~ 1024px)
- 최소 지원: 320px (모바일 폴백)
- 최대 지원: 1280px (대형 태블릿)
- 메뉴 그리드: 태블릿 3~4열, 모바일 2열

## 5. 테스트 요구사항 (Testing)

### TEST-01: 단위 테스트 (Vitest)
- **커버리지 목표**: 80% 이상
- **필수 테스트 대상**:
  - Pinia 스토어 (auth, menu, cart, order, sse)
  - 서비스 레이어 (API 호출, SSE 관리)
  - 유틸리티 함수 (금액 포맷, 검증 로직)
  - 컴포넌트 주요 경로 (렌더링, 이벤트 핸들링)
  - 라우팅 가드

### TEST-02: PBT (fast-check) — 광범위 적용
| 적용 영역 | 테스트 속성 |
|-----------|-------------|
| 장바구니 금액 계산 | 수량 × 단가 합계 항상 정확, 음수 불가 |
| 수량 변경 로직 | 증가/감소 후 수량 ≥ 0, 일관성 유지 |
| Pinia 스토어 상태 전이 | 임의 액션 시퀀스 후 상태 일관성 |
| API 응답 파싱 | 임의 JSON 구조에 대한 안전한 파싱 |
| localStorage 직렬화/역직렬화 | 저장 → 복원 라운드트립 동일성 |
| 라우팅 가드 | 임의 인증 상태 + 라우트 조합에서 올바른 리다이렉트 |

### TEST-03: 컴포넌트 테스트
- Vue Test Utils + Vitest
- 주요 사용자 흐름 테스트 (장바구니 추가 → 주문 확정)
- 에러 상태 렌더링 테스트
- 접근성 테스트 (aria 속성 확인)

## 6. 빌드/배포 요구사항 (Build/Deploy)

### BUILD-01: 번들 최적화 (적극적)
- 라우트별 레이지 로딩 (dynamic import)
- Vite 코드 스플리팅 + 트리 쉐이킹
- CSS purge (Tailwind 미사용 클래스 제거)
- gzip 압축 (Nginx 설정)
- 이미지: 외부 URL이므로 별도 최적화 불필요
- 소스맵: 프로덕션 빌드에서 제외

### BUILD-02: 빌드 환경
- Node.js 20 LTS
- 환경 변수: `VITE_API_BASE_URL` (Backend API 주소)
- 빌드 명령: `npm run build` → `dist/` 생성
- Docker: Nginx로 정적 파일 서빙

## 7. 애니메이션 요구사항 (Animation)

### ANIM-01: 풍부한 애니메이션 적용
| 요소 | 애니메이션 | 지속 시간 |
|------|-----------|-----------|
| 페이지 전환 | 슬라이드 + 페이드 | 200~300ms |
| 모달 열기/닫기 | 스케일 + 페이드 | 200ms |
| 하단 시트 | 슬라이드 업/다운 | 250ms |
| 사이드 드로어 | 슬라이드 좌→우 | 250ms |
| 토스트 알림 | 슬라이드 + 페이드 | 300ms |
| 장바구니 추가 | 바운스/팝 효과 | 300ms |
| 상태 변경 배지 | 펄스/하이라이트 | 500ms |
| 수량 변경 | 숫자 스케일 | 150ms |
| 카테고리 전환 | 페이드 | 150ms |

### ANIM-02: 애니메이션 원칙
- `prefers-reduced-motion` 미디어 쿼리 존중 (접근성)
- GPU 가속 활용 (transform, opacity 위주)
- 60fps 유지 (레이아웃 트리거 속성 회피)
- Vue `<Transition>` 및 `<TransitionGroup>` 활용
