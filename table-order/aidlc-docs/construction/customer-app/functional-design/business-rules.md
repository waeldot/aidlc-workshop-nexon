# Customer App - Business Rules

## 1. 인증 규칙

### AUTH-01: 초기 설정 입력 검증
- 매장 식별자(storeId): 필수, 빈 문자열 불가
- 테이블 번호(tableNumber): 필수, 양의 정수
- 비밀번호(password): 필수, 빈 문자열 불가
- **모든 필드가 유효할 때만 저장 버튼 활성화**

### AUTH-02: 토큰 저장 규칙
- 인증 성공 시 `auth_credentials`와 `auth_token` 모두 localStorage에 저장
- 토큰 형식: JWT (서버에서 발급)
- 저장 시점: 서버 응답 성공 직후

### AUTH-03: 자동 로그인 규칙
- 앱 시작 시 `auth_token` 존재하면 서버 검증 시도
- 검증 실패(401) 시 토큰 삭제 후 credentials로 재인증 시도
- credentials도 실패하면 설정 화면으로 이동

### AUTH-04: 토큰 만료 처리
- API 응답 401 수신 시 즉시 토큰 무효화
- 자동 재인증 시도 (credentials 존재 시)
- 재인증 실패 시 설정 화면 이동 + 에러 메시지

### AUTH-05: 인증 실패 메시지
- 초기 설정 실패: "인증에 실패했습니다. 정보를 확인해 주세요."
- 토큰 만료: "인증 정보가 만료되었습니다. 관리자에게 문의하세요."
- 입력값은 실패 후에도 유지 (재시도 편의)

## 2. 장바구니 규칙

### CART-01: 수량 제한
- 단일 메뉴 최대 수량: **제한 없음** (무제한)
- 최소 수량: 1 (1 미만이면 항목 삭제)

### CART-02: 추가 규칙
- 동일 메뉴 재추가 시 수량 +1 (중복 항목 생성 안 함)
- 추가 성공 시 토스트 피드백 표시

### CART-03: 금액 계산
- `totalAmount = Σ (item.price × item.quantity)`
- `totalQuantity = Σ (item.quantity)`
- 수량/항목 변경 시 즉시 재계산
- 금액 표시 형식: 원화 (예: "15,000원")

### CART-04: localStorage 동기화
- 모든 장바구니 변경 시 즉시 localStorage 동기화
- 앱 시작 시 localStorage에서 복원
- 유효하지 않은 데이터는 빈 장바구니로 초기화

### CART-05: 전체 비우기
- 확인 팝업 필수: "장바구니를 비우시겠습니까?"
- 확인 시 모든 항목 삭제 + localStorage 클리어
- 취소 시 아무 동작 없음

### CART-06: 하단 시트 표시 규칙
- 장바구니 비어있을 때: 접힌 상태 요약 바 숨김 또는 "장바구니가 비어있습니다"
- 장바구니 있을 때: 접힌 상태에서 "{totalQuantity}개 · {totalAmount}원" 표시

## 3. 주문 규칙

### ORDER-01: 주문 전제 조건
- 장바구니가 비어있으면 주문 불가 (주문하기 버튼 비활성화)
- 인증 상태여야 주문 가능

### ORDER-02: 중복 전송 방지
- 주문 확정 버튼 탭 시 즉시 `isSubmitting = true`
- 버튼 비활성화 + 로딩 표시
- 응답 수신 후 `isSubmitting = false`

### ORDER-03: 주문 성공 처리
- 장바구니 즉시 비우기 (localStorage 포함)
- /order/success 화면 이동
- 주문 번호 크게 표시
- 5초 카운트다운 후 /orders (주문 내역) 자동 이동

### ORDER-04: 주문 실패 처리
- 에러 메시지: "주문에 실패했습니다. 다시 시도해 주세요."
- 장바구니 데이터 보존 (삭제하지 않음)
- 주문 확정 버튼 재활성화

### ORDER-05: 주문 데이터 구성
- 필수 포함: storeId, tableId, sessionId, items[], totalAmount
- items[]: menuId, name, quantity, price
- totalAmount는 클라이언트에서 계산하여 전송 (서버에서 재검증)

## 4. SSE 규칙

### SSE-01: 연결 시점
- 인증 성공 직후 SSE 연결 시작
- 엔드포인트: GET /api/sse/customer/{tableId}

### SSE-02: 재연결 정책
- 최대 시도 횟수: **무제한**
- 재연결 간격: 지수 백오프 (1s → 2s → 4s → 8s → 16s → 30s)
- 최대 간격: 30초
- 성공 시 간격 리셋

### SSE-03: 이벤트 처리 규칙
| 이벤트 | 처리 |
|--------|------|
| `order_status_changed` | 해당 주문의 status 업데이트, 배지 색상 변경 |
| `order_deleted` | 해당 주문을 목록에서 제거 |
| `session_terminated` | 장바구니 비우기 + 주문 내역 비우기 + 메뉴 화면 이동 |

### SSE-04: 세션 종료 처리
- 알림 없이 자동 초기화 (조용히 리셋)
- 장바구니 비우기 (localStorage 포함)
- 주문 내역 비우기
- SSE 연결 종료
- 메뉴 화면 (/) 으로 이동

## 5. UI 상태 전이 규칙

### UI-01: 로딩 상태
- API 호출 중 로딩 인디케이터 표시
- 로딩 중 사용자 인터랙션 차단 (해당 영역만)

### UI-02: 에러 상태
- 에러 발생 시 사용자 친화적 메시지 표시
- 내부 에러 정보 노출 금지
- "다시 시도" 버튼 제공 (가능한 경우)

### UI-03: 빈 상태
- 메뉴 없음: "메뉴가 없습니다"
- 장바구니 비어있음: "메뉴를 추가해 주세요"
- 주문 없음: "주문 내역이 없습니다"

### UI-04: 주문 상태 배지 색상
| 상태 | 색상 | 텍스트 |
|------|------|--------|
| pending | 노란색 (#FFC107) | 대기중 |
| preparing | 파란색 (#2196F3) | 준비중 |
| completed | 초록색 (#4CAF50) | 완료 |

### UI-05: 터치 영역
- 모든 인터랙티브 요소: 최소 44×44px 터치 영역
- 버튼, 탭, 카드 등 모두 적용

## 6. 네비게이션 규칙

### NAV-01: 사이드 드로어
- 열기: 햄버거 아이콘 탭 또는 왼쪽 스와이프
- 닫기: 배경 탭, X 버튼, 오른쪽 스와이프
- 메뉴 항목: 메뉴(/), 주문 내역(/orders)
- 테이블 정보 표시: "테이블 {tableNumber}"

### NAV-02: 라우트 가드
- 인증 필요 페이지: 미인증 시 /setup 리다이렉트
- /setup: 이미 인증됨이면 / 리다이렉트
- /order/confirm: 장바구니 비었으면 / 리다이렉트
- /order/success: 주문 결과 없으면 / 리다이렉트

### NAV-03: 하단 시트 (장바구니)
- 메뉴 화면에서만 표시
- 접힌 상태가 기본
- 위로 스와이프 또는 탭으로 펼치기
- 아래로 스와이프 또는 핸들 탭으로 접기

## 7. 데이터 검증 규칙

### VAL-01: 입력값 검증 (클라이언트)
- XSS 방지: 사용자 입력 HTML 이스케이프
- 숫자 필드: 양의 정수만 허용
- 문자열 필드: 앞뒤 공백 제거 (trim)

### VAL-02: API 응답 검증
- 응답 구조 검증 (예상 필드 존재 확인)
- 타입 불일치 시 에러 처리
- null/undefined 안전 처리

### VAL-03: localStorage 데이터 검증
- 읽기 시 JSON 파싱 에러 처리
- 스키마 불일치 시 기본값으로 초기화
- 손상된 데이터 감지 시 삭제 후 재설정
